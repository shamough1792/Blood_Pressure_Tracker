# 血壓記錄系統 - 長者友善介面 redesign

## 緣由

原系統輸入流程需要填寫 5 個欄位（高壓、低壓、心跳、日期、時間），對年長使用者來說步驟繁瑣。本 redesign 目標是大幅簡化輸入流程，同時優化檢視記錄的體驗。

## 使用者習慣

- 每天固定量兩次血壓（早上、晚上各一次）
- 使用傳統手臂式血壓計，手動輸入數字
- 使用裝置：手機 或 電腦
- 需要檢視歷史記錄和匯出 Excel

## 設計方案

採用**方案二：兩步驟引導輸入**（逐步對話式），搭配**卡片式記錄列表**。

---

## 1. 輸入流程：4 步驟引導

### 核心邏輯
- **日期**：自動填入今天，長輩不需填寫
- **時段**：根據當前時間自動判斷（早上 5:00~12:00 = 上午，12:00~ = 下午）
- 長輩只需輸入 3 個數字：高壓、低壓、心跳

### 步驟 1/4 — 輸入高壓
- 進度條顯示當前位置（● ● ● ○  步驟 1/4）
- 大字標題：「高壓是多少？」
- 超大數字輸入框（placeholder: ___）
- 手機上數字鍵盤自動彈出
- 「下一步」綠色大按鈕
- 驗證範圍 50~250

### 步驟 2/4 — 輸入低壓
- 同上，標題改為「低壓是多少？」
- 左上角「← 上一步」按鈕
- 驗證範圍 30~150

### 步驟 3/4 — 輸入心跳
- 同上，標題改為「心跳是多少？」
- 驗證範圍 30~200

### 步驟 4/4 — 確認提交
- 顯示所有資料摘要：高壓、低壓、心跳、日期、時段
- **血壓分級顏色提示**：
  - 🟢 綠色（正常）：高壓 <120 且 低壓 <80
  - 🟡 黃色（偏高）：高壓 120~139 或 低壓 80~89
  - 🔴 紅色（高血壓）：高壓 ≥140 或 低壓 ≥90
- 「✓ 確認儲存」大按鈕

### 提交成功
- 顯示成功訊息，3 秒後自動回到步驟 1
- 按鈕「再記一筆」可手動返回

---

## 2. 查看記錄：卡片式列表

### 月份切換
- 大字顯示當前月份（如「2026年 5月」）
- 左右箭頭按鈕「〈」和「〉」切換月份
- 取代原本的下拉選單

### 卡片設計
每筆記錄顯示為獨立卡片：
- 日期 + 星期 + 時段（如「5月2日 (六) 上午」）
- 高壓數值、低壓數值
- 心跳數值
- 血壓狀態標籤（🟢 正常 / 🟡 偏高 / 🔴 高血壓）
- [修改] [刪除] 按鈕（大字）

### 匯出 Excel
- 頂部導航列保留「📥 匯出 Excel」按鈕
- 功能完全沿用現有邏輯

---

## 3. 修改記錄頁面

- 大字版表單（字體 24px+）
- 欄位：高壓、低壓、心跳、時段選擇（上午/下午）、日期
- 「✓ 儲存修改」大按鈕
- 「← 返回記錄」連結

---

## 4. 響應式設計

### 手機版
- 每個步驟佔滿螢幕高度
- 輸入框置中，觸控友善
- 數字鍵盤自動彈出
- 按鈕在下方拇指可及範圍

### 電腦版
- 內容置中，寬度不超過 500px
- 字體 28px+，按鈕高度 60px+
- 支援鍵盤 Enter 快速跳下一步

---

## 5. 技術修改範圍

| 檔案 | 改動 |
|------|------|
| `views/index.ejs` | 完全重寫為 4 步驟引導流程（JavaScript 切換步驟） |
| `views/records.ejs` | 完全重寫為卡片式列表 + 月份左右切換 |
| `views/modify.ejs` | 改為大字版表單 |
| `public/styles.css` | 全部重新設計（大字體、卡片、顏色標示、響應式） |
| `server.js` | 日期/時段自動填入邏輯；不動既有 CRUD 和匯出 |
| `package.json` | 不變動 |

### 不受影響的功能
- `/add` POST 路由 — 完全沿用
- `/records` GET 路由 — 資料查詢邏輯不變，只改 view
- `/modify/:id` 和 `/update/:id` — 沿用
- `/delete/:id` — 沿用
- `/export/excel` — 沿用
